Production Process

A streamlined guide for the standard Git workflow to ensure code quality and stable releases.

01
Setup

Initialize Project

Create a new GitHub repository to host your project's source code and version history.

02
Production

Main Branch

Establish the main branch. This branch always contains the current production-ready code.

03
Staging

QA / Develop / Staging

Create a qa or develop branch from main for integration and testing.

04
Development

Your Branch

Branch out from qa to your local feature branch. Work on isolated changes here.

05
Sync

Push Code

Once your feature is complete, push your local commits to your remote feature branch.

06
Integration

Merge to QA

Open a Pull Request and merge your branch into qa. Resolve any conflicts during this phase.

07
Verification

Quality Control

Thoroughly test all features and functionality in the QA environment. Verify everything works as expected.

Does it work perfectly?

YES (Step 08)

Go Live

Merge qa into main. Deploy the stable release to the production environment.

NO (Step 09)

Fix Bugs

Fix the issues on your branch and repeat steps 5 to 8 until verification passes.
